KEY FINDINGS
================================================================================

OFFICIAL API:
- PumpPortal is THE primary third-party API (not official Pump.fun)
- Two API modes: Lightning (auto-execute) and Local (self-signed)
- WebSocket streaming available for real-time events
- 0.5% trading fee per transaction

CORE ENDPOINTS:
- Trading: https://pumpportal.fun/api/trade-local
- Lightning: https://pumpportal.fun/api/trade?api-key=KEY
- WebSocket: wss://pumpportal.fun/api/data
- Bonding Curve: https://rpc.api-pump.fun/bondingCurve

AUTHENTICATION:
- API key required from PumpPortal
- Private key handling: NEVER transmit to API
- Always use Local Transaction API for production
- Sign transactions locally only

TRADING OPERATIONS:
- Buy: Denominated in SOL, purchase tokens
- Sell: Denominated in tokens, receive SOL
- Slippage: Critical parameter (typical 10-50%)
- Error code 6002: Most common (slippage exceeded)

BONDING CURVE:
- Token launches at 0% bonding
- Price increases with each purchase
- Auto-migrates to Raydium at 100% bonding
- Creator receives 0.5 SOL on completion

FEE STRUCTURE:
- Trading fee: 0.5% (calculated before slippage)
- Creator incentive: 0.5 SOL at 100% bonding
- Network fees: ~0.00005 SOL typical

SECURITY:
- Use Local Transaction API (CRITICAL)
- Store keys in secure vault (AWS KMS, etc.)
- Never hardcode private keys
- Implement transaction simulation
- Sign locally only
- Rate limit WebSocket connections

SDKS & LIBRARIES:
- Official: @pump-fun/pump-sdk, @pump-fun/pump-swap-sdk
- Community: Synthexlab/pumpfun-sdk, @cryptoscan/pumpfun-sdk
- Base: @solana/web3.js, @solana/spl-token

ERROR HANDLING:
- 6002: TooMuchSolRequired (slippage)
- 6004: InsufficientSOL (balance)
- 6003: MintTooBig (amount)
- 0x16: InsufficientTokens
- Most require retry or parameter adjustment

GOTCHAS & LIMITATIONS:
- No official Pump.fun API (use third-party)
- WebSocket blacklisting for aggressive connections
- Specific rate limits not publicly documented
- Simulation costs ~0.00014 SOL
- Transaction may fail after signing
- Bonding curve can close unexpectedly

RECOMMENDED APPROACH:
1. Use Local Transaction API exclusively
2. Implement comprehensive error handling
3. Simulate before execution
4. Use single persistent WebSocket
5. Monitor transaction success rates
6. Set spending limits
7. Test thoroughly on devnet
8. Deploy with safeguards
